Общие остатки можно загружать через API
24 августа 2022

Если ваши товары физически хранятся на одном складе, а на Маркете продаются с нескольких, объедините такие склады в группу. Недавно рассказывали, как это упрощает ручную работу с остатками по разным моделям — а теперь упростит и автоматическую.

Объединить склады

  • Остатки будут синхронно обновляться и уменьшаться при продажах, так что вы защититесь от заказов-дублей и вынужденных отмен.
  • Перенастраивать API не придётся: за данными будем приходить через уже существующий запрос к одному из складов группы.

Как настроить общие остатки по API

  • На странице Логистика → Склады нажмите кнопку «Создать группу».
  • Выберите склады с методом работы «через API». После объединения их товары временно пропадут с витрины — это необходимо, чтобы остатки стали общими.
  • Перенастраивать API не придётся: за данными будем приходить через уже существующий запрос к одному из складов группы. Выберем его сами и покажем на странице Настройки API.
  • Товары вернутся на витрину с ближайшим обновлением остатков. После этого при продажах их количество будет синхронно уменьшаться у всей группы.

Чтобы не ждать ближайшего обновления и быстрее выйти на витрину, можете после создания группы передать общие остатки через запрос магазина к Маркету. Вот как он настраивается.
